# 무인(밤샘) 자율 구현 프로토콜

`/goal`로 4개 구현 계획을 사람 없이 순차 실행할 때 따르는 규칙. 이 파일을 읽고 그대로 따른다.

## 목표

`docs/superpowers/plans/`의 4개 계획을 **순서대로(1 → 2 → 3 → 4)** 구현한다.
계획 N의 모든 태스크가 그린(테스트·타입체크 통과)이고 커밋되기 전에는 계획 N+1을 시작하지 않는다.

- 계획 1: `2026-05-26-plan-1-foundation.md`
- 계획 2: `2026-05-26-plan-2-tracks-listening.md`
- 계획 3: `2026-05-26-plan-3-record-sequencer.md`
- 계획 4: `2026-05-26-plan-4-play-scoring.md`

실행은 **executing-plans 스킬**로 태스크 단위 진행한다.

## 태스크 진행 규칙

1. 계획에 적힌 bite-sized TDD 스텝(실패 테스트 → 실패 확인 → 구현 → 통과 확인)을 그대로 따른다.
2. 태스크 종료 시 반드시 `npm run test:run && npx tsc -b`를 실행해 **둘 다 통과**를 확인한다.
3. 그린이면 계획에 적힌 커밋 메시지로 커밋한다. (커밋 메시지 끝에 `Co-Authored-By: Claude Opus 4.7 (1M context) <noreply@anthropic.com>` 포함)
4. 푸시는 각 계획 완료 시점마다 한다.

## 막혔을 때 (가장 중요)

- 단위 테스트/타입체크 실패 시 **systematic-debugging 스킬**로 원인을 찾아 고친다.
- 합리적인 시도(같은 문제에 2~3회) 후에도 해결 안 되거나, **계획 내용이 모호해 판단이 필요하면 추측하지 말고 멈춘다.**
- 멈출 때는 `IMPLEMENTATION_NOTES.md`에 상황을 기록한다: 어느 계획/태스크, 무슨 문제, 시도한 것, 사람에게 필요한 결정.

## 브라우저 수동 검증 스텝

- 무인 환경에서는 실제 브라우저 동작(소리 동기·캔버스 렌더·키 입력 등)을 **진짜로 확인할 수 없다.**
- 할 수 있는 것(빌드, `tsc`, 단위 테스트)만 하고, **성공했다고 꾸미지 않는다.**
- 각 "수동 검증" 스텝은 `IMPLEMENTATION_NOTES.md`의 "사람 검증 필요" 목록에 항목으로 남긴다(어느 계획의 어떤 확인 항목인지).

## 진행상황 기록

- `IMPLEMENTATION_NOTES.md`를 진행 로그로 사용한다. 태스크 완료/커밋 해시/막힌 지점/사람 검증 필요 항목을 누적 기록한다.
- 아침에 사람이 이 파일 하나만 봐도 밤새 무슨 일이 있었는지 파악되도록 한다.

## 완료 정의

`docs/superpowers/plans/`의 4개 계획이 모두 구현되고, `npm run test:run`과 `npx tsc -b`가 통과하며, 남은 "사람 검증 필요" 항목이 `IMPLEMENTATION_NOTES.md`에 정리되어 있으면 완료.
