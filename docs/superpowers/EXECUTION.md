# 무인(밤샘) 자율 구현 프로토콜 — Editor v2

`/goal`로 Editor v2 계획을 사람 없이 순차 실행할 때 따르는 규칙. 이 파일을 읽고 그대로 따른다.

> v1(제품 기반: 계획 1~4 `2026-05-26-plan-*`)은 구현 완료됨. 이 프로토콜은 **Editor v2 재편**을 대상으로 한다.

## 목표

`docs/superpowers/plans/`의 **v2 계획 6개를 순서대로(1 → 2 → 3 → 4 → 5 → 6)** 구현한다.
계획 N의 모든 태스크가 그린(테스트·타입체크 통과)이고 커밋되기 전에는 계획 N+1을 시작하지 않는다.

- 계획 1: `2026-05-27-plan-v2-1-viewport-timeline.md` — 공유 뷰포트 + 타임라인 골격 (요구 10)
- 계획 2: `2026-05-27-plan-v2-2-track-rows.md` — 행 분해 + 마커 에디터 + 포커스 확장 (요구 5·9·11)
- 계획 3: `2026-05-27-plan-v2-3-track-editor-controls.md` — 상태그리드·볼륨팝오버·키캡·마커비우기 (요구 1·2·3·7)
- 계획 4: `2026-05-27-plan-v2-4-dnd-reorder.md` — 트랙 드래그 정렬 + 영속 (요구 8)
- 계획 5: `2026-05-27-plan-v2-5-toolbar-sequencer.md` — 전역 툴바 + 인라인 시퀀서 (요구 4)
- 계획 6: `2026-05-27-plan-v2-6-keyboard.md` — 키보드 차단 + 재생키 바인딩 (요구 6·12)

**공유 계약(단일 기준):** `docs/superpowers/plans/2026-05-27-v2-contracts.md` — 파일구조·타입·시그니처·테스트 규약. 계획과 어긋나는 부분이 있으면 **계약을 우선**한다.

**설계 출처:** `docs/superpowers/specs/2026-05-27-editor-architecture-v2-design.md`

**브랜치:** `feat/editor-v2`. 푸시는 이 브랜치로 한다.

실행은 **executing-plans 스킬**(이 세션 내 배치 실행) 또는 **subagent-driven-development 스킬**(태스크별 새 서브에이전트 + 리뷰)로 태스크 단위 진행한다.

## 태스크 진행 규칙

1. 계획에 적힌 bite-sized TDD 스텝(실패 테스트 → 실패 확인 → 구현 → 통과 확인)을 그대로 따른다.
2. 태스크 종료 시 반드시 `yarn test:run && yarn tsc -b`를 실행해 **둘 다 통과**를 확인한다.
3. 그린이면 계획에 적힌 커밋 메시지로 커밋한다. (커밋 메시지 끝에 `Co-Authored-By: Claude Opus 4.7 (1M context) <noreply@anthropic.com>` 포함)
4. 푸시는 각 계획 완료 시점마다 `feat/editor-v2`로 한다.

## 막혔을 때 (가장 중요)

- 단위 테스트/타입체크 실패 시 **systematic-debugging 스킬**로 원인을 찾아 고친다.
- 합리적인 시도(같은 문제에 2~3회) 후에도 해결 안 되거나, **계획/계약 내용이 모호해 판단이 필요하면 추측하지 말고 멈춘다.**
- 멈출 때는 `IMPLEMENTATION_NOTES.md`에 상황을 기록한다: 어느 계획/태스크, 무슨 문제, 시도한 것, 사람에게 필요한 결정.

## 브라우저 수동 검증 스텝

- 무인 환경에서는 실제 브라우저 동작(캔버스/SVG 렌더·가로 스크롤/줌·드래그·포커스 애니메이션·소리·키 입력 등)을 **진짜로 확인할 수 없다.**
- 헤드리스 Chrome 드라이버(`/tmp/bof-driver`, `yarn dev` 후 `chrome --headless --screenshot`)로 가능한 만큼 시각 확인하되, 드래그/소리처럼 스크린샷으로 못 잡는 것은 무리하지 않는다.
- 할 수 있는 것(빌드, `tsc`, 단위 테스트, 스크린샷)만 하고, **성공했다고 꾸미지 않는다.**
- 각 "브라우저 검증" 스텝은 `IMPLEMENTATION_NOTES.md`의 "사람 검증 필요" 목록에 항목으로 남긴다(어느 계획의 어떤 확인 항목인지).

## 진행상황 기록

- `IMPLEMENTATION_NOTES.md`를 진행 로그로 사용한다. 태스크 완료/커밋 해시/막힌 지점/사람 검증 필요 항목을 누적 기록한다.
- 아침에 사람이 이 파일 하나만 봐도 밤새 무슨 일이 있었는지 파악되도록 한다.

## 완료 정의

v2 계획 6개가 모두 구현되고, `yarn test:run`과 `yarn tsc -b`가 통과하며, 남은 "사람 검증 필요" 항목이 `IMPLEMENTATION_NOTES.md`에 정리되어 `feat/editor-v2`에 푸시되어 있으면 완료.
