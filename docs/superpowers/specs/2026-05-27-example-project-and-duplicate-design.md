# 예제 프로젝트 · 프로젝트 복사 · 이름 수정 설계

작성일: 2026-05-27

## 목표

1. **예제 프로젝트** — 데모 베이스플로(`public/samples/moodmode-demo.mp3`)를 오프라인 음악
   분석으로 골고루 마커링한 쇼케이스 프로젝트를, 목록 화면의 secondary 버튼 클릭 시 새 사본으로 생성.
2. **프로젝트 복사** — 기존 프로젝트를 자산(Blob)까지 안전하게 복제.
3. **이름 수정** — 목록 카드와 에디터 상단 둘 다에서 프로젝트 이름 인라인 편집.

## 배경 / 제약

- 클라이언트 전용. 오디오는 경로 참조가 아니라 **Blob을 IndexedDB `assets` 스토어에 복사**하고,
  프로젝트는 `baseFlow.assetId`로 그 자산을 참조한다 (`src/persistence/assets.ts`,
  `src/ui/ProjectList.tsx`). 따라서 복제·예제 생성 모두 **자산 blob을 새로 만들어야** id 공유로
  인한 삭제 깨짐이 없다.
- 빌드/배포 환경에 ffmpeg 없음. 분석에 필요한 PCM은 wasm 디코더(`mpg123-decoder`, **devDependency**)로
  오프라인에서만 얻고, 결과(마커)는 정적 모듈로 커밋한다. 런타임/일반 빌드는 디코더에 의존하지 않는다.
- 데모 사실: 48000 Hz, 약 136.03초, 256kbps CBR MP3 (node 디코딩으로 확인).

## 1. 오프라인 음악 분석 — `scripts/analyze-demo.mjs`

기존 `scripts/gen-samples.mjs`와 동일한 "오프라인 생성 → 정적 데이터 커밋" 컨벤션을 따른다.
실행: `node scripts/analyze-demo.mjs` → `src/example/exampleData.generated.ts` 갱신.

파이프라인:

1. `mpg123-decoder`로 PCM 디코딩 → 모노 믹스다운 → 약 22050 Hz로 다운샘플.
2. STFT: Hann 창, 크기 1024, hop 512. 스크립트 내에 무의존 radix-2 FFT 구현.
3. 3개 주파수 대역으로 분리해 대역별 **스펙트럴 플럭스 온셋 엔벨로프**(양의 스펙트럼 증가량 합) 계산.
   - 저역(킥): ~20–150 Hz
   - 중역(스네어/클랩/톰): ~150–2000 Hz
   - 고역(하이햇/퍼커션): ~4000–16000 Hz
4. 대역별 피크 피킹: 이동평균 임계 + 최소 간격으로 온셋 시각 확정.
5. 템포 추정: 전대역 온셋 엔벨로프 자기상관 → BPM(90–160 범위로 접기). 위상 = 첫 강 온셋.
6. 온셋을 16분음표 그리드(beat/4)에 양자화 → 중복 제거.
7. 6개 샘플로 라우팅(주 타격 + 부 타격 분리, 트랙당 밀도 상한):
   - 저역 → `kick`(다운비트 위주), `tom`(드문 저-중역 강세)
   - 중역 → `snare`(백비트), `clap`(강세 액센트)
   - 고역 → `hat`(8/16분), `perc`(싱코페이션)
   - "골고루": 6개 트랙 모두 비지 않도록 각 트랙 최소 마커 수 보장.

출력 모듈 형태(생성 파일, 수정 금지 주석 포함):

```ts
// AUTO-GENERATED by scripts/analyze-demo.mjs. Do not edit by hand.
export const EXAMPLE_DURATION_MS = 136032;
export const EXAMPLE_BPM = 124; // 추정값(참고용)
export const EXAMPLE_TRACK_MARKERS: Record<string, number[]> = {
  kick: [/* ms */],
  snare: [...],
  hat: [...],
  clap: [...],
  tom: [...],
  perc: [...],
};
```

순수 분석 헬퍼(피크 피킹, 양자화, 대역 에너지)는 스크립트가 아니라 테스트 가능한 모듈
(`src/example/analysis.ts` 또는 스크립트가 import하는 순수 함수 모음)로 분리해 합성 신호로 단위 테스트한다.

## 2. 예제 프로젝트 블루프린트 & 생성 — `src/example/exampleProject.ts`

청사진은 id/assetId가 없는 정적 정의이며, 생성된 마커 데이터를 결합한다.

```ts
interface ExampleTrackBlueprint {
  name: string;
  status: TrackStatus;          // "listening"
  sound: SoundRef;              // { kind: "builtin", sampleId }
  keyBinding: string | null;
  volume: number;
  color: string;
  markersMs: number[];          // EXAMPLE_TRACK_MARKERS[sampleId]
}
interface ExampleBlueprint {
  name: string;                 // "예제 프로젝트"
  master: { volume: number };
  durationMs: number;           // EXAMPLE_DURATION_MS (안전상 디코딩 값으로 덮어씀)
  tracks: ExampleTrackBlueprint[];
}
```

순수 함수 `buildProjectFromBlueprint(blueprint, assetId, durationMs): Project`:

- `id`, 각 트랙 `id`, 각 마커 `id`에 `newId()` 부여.
- `createdAt = updatedAt = Date.now()`.
- `baseFlow = { kind: "audioFile", assetId, durationMs }`.
- `transport = { playPauseKey: null }`.
- 이 함수는 fetch/decode와 무관해 단위 테스트 가능.

생성 흐름(ProjectList의 `createExampleProject`):

1. `fetch("/samples/moodmode-demo.mp3")` → `blob()`.
2. `putAsset(blob, "moodmode-demo.mp3")` → `assetId`.
3. `getEngine().decode(blob)` → `durationMs = buffer.duration * 1000`.
4. `buildProjectFromBlueprint(EXAMPLE_BLUEPRINT, assetId, durationMs)`.
5. `saveProject(project)` → `setProject(project)` → `onOpen(project)` (에디터 진입).

매 클릭 **항상 새 사본**(이미 있어도 새로 생성). 키바인딩은 6개 트랙에 기본키(예: A,S,D,F,G,H) 부여.

## 3. 프로젝트 복사

`src/persistence/assets.ts`:

```ts
export async function copyAsset(id: string): Promise<string>;
// getAsset(id) → 없으면 throw, 있으면 putAsset(asset.blob, asset.name) → 새 id
```

`src/persistence/projects.ts`:

```ts
export async function duplicateProject(project: Project): Promise<Project>;
```

- 깊은 복제(`structuredClone`).
- `baseFlow.assetId = await copyAsset(원본)`.
- 각 `track.sound.kind === "upload"`이면 `track.sound.assetId = await copyAsset(원본)`.
- `id`, 트랙 `id`, 마커 `id` 전부 `newId()` 재부여.
- `name = `${project.name} (사본)``, `createdAt = updatedAt = Date.now()`.
- `saveProject(copy)` 후 반환.

UI: 목록 카드 푸터 "복사" 버튼 → `duplicateProject` → 목록 갱신(자동 진입 안 함).

## 4. 이름 수정

- **에디터 상단** (`src/ui/Editor.tsx` `top-bar__name`): 클릭 시 인라인 `input`으로 전환,
  Enter/blur 커밋 → store `renameProject(name)`(기존 액션, `updatedAt` 갱신) → autosave가 영속화.
  Esc 취소.
- **목록 카드** (`src/ui/ProjectList.tsx`): 이름 옆 연필 아이콘 → 인라인 `input` →
  커밋 시 `saveProject({ ...p, name, updatedAt: Date.now() })` + 목록 갱신. 빈 이름은 거부(원복).

## 사람 검증 필요

- 생성된 마커가 실제 청취 시 음악적으로 들어맞는지(헤드리스 한계). 양자화/대역 라우팅이
  과밀/과소하면 스크립트 파라미터(임계, 최소간격, 밀도 상한, 대역 경계) 조정.
- 인라인 이름 편집 포커스/키 동작이 에디터의 전역 키 핸들러(play/record)와 충돌하지 않는지.
